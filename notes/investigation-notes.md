# Investigation Notes

## Objective

The objective of this week's labs was to understand how detection engineering supports SOC operations by developing detections, validating alerts, correlating security events, and mapping attacker behaviors to the MITRE ATT&CK Framework.

---

## Key Investigation Findings

### Detection Engineering Fundamentals

- Learned how detections are developed from attacker behaviors rather than malware signatures.
- Understood the importance of Windows Security Logs and Sysmon telemetry.
- Explored the role of Sigma rules in creating vendor-independent detections.

---

### Certutil Detection

- Identified abuse of Certutil as a Living-off-the-Land Binary (LOLBin).
- Developed detection logic for suspicious Certutil command-line usage.
- Recognized the importance of correlating Certutil activity with process creation and network connections.

---

### RDP Investigation

- Investigated repeated failed logins followed by successful RDP authentication.
- Distinguished between legitimate administrative access and suspicious remote logons.
- Evaluated authentication events within the context of user behavior.

---

### PowerShell Investigation

- Investigated encoded PowerShell execution.
- Recognized common attacker techniques involving obfuscated commands.
- Assessed how PowerShell logging improves endpoint visibility.

---

### False Positive Analysis

- Evaluated alerts generated by legitimate administrative activity.
- Learned to distinguish benign behavior from malicious activity using context.
- Understood the importance of detection tuning to reduce unnecessary alerts.

---

### MITRE ATT&CK Mapping

Mapped detections to the ATT&CK Framework, including:

- T1110 – Brute Force
- T1021.001 – Remote Desktop Protocol
- T1059.001 – PowerShell
- T1218.010 – Certutil
- T1547.001 – Registry Run Keys
- T1053.005 – Scheduled Task

---

## Overall Findings

Current detections provide strong coverage for:

- Execution
- Persistence
- Credential Access
- Defense Evasion

Additional improvements are recommended for:

- Initial Access
- Lateral Movement
- Command and Control
- Exfiltration

---

## Conclusion

Detection engineering is an iterative process that combines endpoint telemetry, behavioral analysis, detection logic, and continuous tuning. Mapping detections to MITRE ATT&CK provides a standardized method for measuring detection coverage and improving SOC visibility.
